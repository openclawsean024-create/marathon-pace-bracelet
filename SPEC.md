## 1. 市場研究

### 1.1 市場概況

**全球跑步市場**
- 2024 年全球跑步人口：約 3 億人
- 馬拉松賽事數量持續成長，台灣每年舉辦超過 20 場
- 智慧穿戴裝置普及，配速需求增加

**目標市場現況**
- 跑者需要配速參考，手機查詢不便
- 現有配速工具多為網頁版，缺乏離線存取
- 需要視覺化配速計畫

### 1.2 競爭對手分析

| 競品 | 目標市場 | 核心功能 | 定價 | 劣勢 |

## 5. 配速策略完整說明

### 5.1 配速策略總覽

配速是馬拉松成敗的關鍵。本產品支援以下五種核心配速策略：

| 策略名稱 | 適用情境 | 體力消耗模式 | 風險 |
|---------|---------|------------|------|
| **均速法（Even Pace）** | 初次跑馬、有經驗的穩定跑者 | 平均分配 | 低 |
| **負分段（Negative Split）** | 有基礎想突破 PB | 前慢後快 | 中 |
| **正分段（Positive Split）** | 體力有限、安全完賽為主 | 前快後慢 | 高（後段崩潰） |
| **敵我配速（Adversarial Pace）** | 針對特定對手或目標設定 | 動態調整 | 中 |
| **心率區間配速（HR Zone Pace）** | 有心率錶數據的科學派跑者 | 按心率區間調配 | 低 |

---

### 5.2 均速法（Even Pace）

**原理**：全程維持相同的配速，是最科學也最安全的策略。

**適合對象**：
- 首次參加全馬的跑者
- 體力分配觀念尚未建立的初學者
- 目標是「安全完賽」而非「創最佳成績」

**執行方式**：
```
目標時間：4:30:00
平均配速：6:25 /km

每公里配速固定為 6:25
累計時間：6:25 → 12:50 → 19:15 → ... → 4:30:00
```

**優點**：
- 避免前半段過快導致「撞牆期」
- 體力消耗均勻，不易在後段崩潰
- 心理壓力小，節奏穩定

**缺點**：
- 對於有實力的跑者，可能未能發揮全部潛力
- 在有坡度的賽道難以維持精確均速

---

### 5.3 負分段（Negative Split）

**原理**：前半馬稍微保守（配速較慢），後半馬加速（配速較快）。典型的「前慢後快」策略。

**適合對象**：
- 有半年以上系統訓練的跑者
- 目標是突破個人最佳成績（PB）
- 了解自己體力極限的中級以上跑者

**執行方式**（以 4:00:00 完賽為例）：

| 段落 | 配速調整 | 範例配速 | 說明 |
|------|---------|---------|------|
| **1～10K** | -10～15秒/km | 5:50～5:55 | 熱身期，刻意压慢，保存體力 |
| **10～21K** | -5秒/km | 5:45 | 建立節奏，进入巡航狀態 |
| **21～30K** | ±0秒/km | 5:40 | 維持配速，保持專注 |
| **30～35K** | +5秒/km | 5:35 | 開始加速，測試體力 |
| **35～42K** | -10秒/km | 5:25 | 最後衝刺，燃燒剩餘體力 |

**進階負分段（馬太效應）**：
```
前 21K：比平均配速慢 10-15 秒/km
後 21K：比平均配速快 10-15 秒/km
最終：後半馬比前半馬快 30 秒～2 分鐘
```

**優點**：
- 最大化脂肪燃燒比率（前期有氧供給稳定）
- 避免血糖急降導致的「撞牆」
- 後段加速能帶來心理優勢

**缺點**：
- 需要紀律，前半段不能因為興奮而跑太快
- 需要對自己的能力有正確認知

---

### 5.4 正分段（Positive Split）

**原理**：前半馬配速較快，後半馬逐漸放慢。是業餘跑者最常見但最不推薦的模式。

**適合對象**：
- 體力有限，目標是安全完賽
- 不追求成績，只求完賽的休閒跑者
- 已被診斷有健康風險的跑者（保守策略）

**警告**：正分段是大多數業餘跑者後段崩潰的主因。研究顯示，85% 的業餘跑者採用正分段，其中 60% 在 30K 後明顯掉速。

**典型後段掉速模式**：
```
1～21K：目標配速 5:30/km
22～30K：掉速至 5:45/km
31～35K：掉速至 6:00/km
36～42K：掉速至 6:30/km 或更慢
→ 最終比前半馬慢 10-20 分鐘
```

**為什麼不推薦：**
1. 前半段血糖和肝醣充沛，容易跑太快
2. 乳酸在肌肉中堆積，配速自然下降
3. 心理上「已經跑完一半」的錯覺導致鬆懈

---

### 5.5 敵我配速（Adversarial Pace）

**原理**：根據對手的實力、賽道特性、氣候條件動態調整配速，常用於有競爭對手或設定「分段目標」的情境。

**適合對象**：
- 有明確對手或成績目標的競技型跑者
- 想要「跟著某個兔子（配速員）」跑的跑者
- 賽道有明確分段目標（如折返點、配速切換點）

**應用情境**：

| 情境 | 配速策略 |
|------|---------|
| **有配速員（兔子）** | 跟隨兔子的配速，在 35K 後評估體力決定是否超越 |
| **分段时间目標** | 設定每 10K 的分段目標時間，如 40K 前不低於 5:45/km |
| **對手追擊** | 前 30K 故意壓慢 5 秒/km，保留體力在最後 12K 追擊 |
| **天氣應變** | 高溫時段（10:00-14:00）主動放慢 10-15 秒/km |

**實作範例**（4:00:00 完賽，分 5 段策略）：
```
第一段（0-10K）：5:40/km ─ 穩定起動，建立位置
第二段（10-21K）：5:35/km ─ 巡航配速，保持專注
第三段（21-30K）：5:40/km ─ 體力高原期，保守策略
第四段（30-38K）：5:30/km ─ 逐步加速，拉開差距
第五段（38-42K）：5:20/km ─ 全力衝刺
```

---

### 5.6 心率區間配速（HR Zone Pace）

**原理**：根據跑者的心率區間設定配速，確保在正確的強度區間運行。

**心率區間定義**（依據最大心率 HRmax）：

| 區間 | 名稱 | 心率範圍 | 配速特性 |
|------|------|---------|---------|
| **Z1** | 暖身區 | 50-60% HRmax | 恢復跑，無法用於比賽 |
| **Z2** | 脂肪燃燒區 | 60-70% HRmax | 馬拉松主力區，長距離耐力跑 |
| **Z3** | 有氧區 | 70-80% HRmax | 半馬到全馬的臨界配速 |
| **Z4** | 無氧區 | 80-90% HRmax | 只能維持 1-2 小時，全馬後段 |
| **Z5** | 極限區 | 90-100% HRmax | 只能維持數分鐘，衝刺用 |

**實作方式**：
1. 用戶輸入最大心率（或安靜心率，系統自動估算最大心率）
2. 系統自動計算五個心率區間
3. 根據策略自動分配每段的心率目標
4. 用戶需配戴心率裝置對照（不支援自動抓取，須手動對照）

**馬拉松配速對照表**（以最大心率 185 為例）：

| 段落 | 建議心率區間 | 配速範例 |
|------|------------|---------|
| 0-10K | Z2 (111-130 bpm) | 6:00/km |
| 10-21K | Z2-Z3 (130-148 bpm) | 5:45/km |
| 21-30K | Z3 (148-148 bpm) | 5:40/km |
| 30-38K | Z3-Z4 (148-166 bpm) | 5:30/km |
| 38-42K | Z4 (166-185 bpm) | 5:15/km |

### 5.7 配速策略選擇決策樹

```
你的馬拉松目標是什麼？
│
├─ 安全完賽，不想受傷
│   └─ → 均速法（Even Pace）
│
├─ 突破 PB，有系統訓練
│   │
│   ├─ 有心率裝置
│   │   └─ → 心率區間配速
│   │
│   └─ 無心率裝置
│       └─ → 負分段（Negative Split）
│
├─ 有明確對手或分段目標
│   └─ → 敵我配速（Adversarial Pace）
│
└─ 第一次跑，不確定自己能力
    └─ → 均速法 + 每 5K 檢視調整
```

## 6. 列印功能規格（PWA 列印支援）

### 6.1 列印功能需求背景

許多跑者習慣將配速計劃列印出來攜帶到賽道上使用（尤其在無法使用手機的情境）。因此本產品需支援高質量的列印輸出。

### 6.2 列印模式功能規格

#### 6.2.1 專用列印版面設計

| 元素 | 規格 |
|------|------|
| **紙張格式** | A4 / Letter（自動偵測） |
| **字體大小** | 標題 18pt、內容 10pt、公里分段 12pt |
| **字體種類** | Noto Sans TC（列印時嵌入） |
| **邊界** | 左右 1.5cm、上下 1cm |
| **QR Code** | 右上角，掃描後可打開手機版連結 |

#### 6.2.2 列印內容區塊

```
┌──────────────────────────────────────────┐
│  馬拉松配速手環 - 配速計劃表        [QR] │
│  ──────────────────────────────────      │
│  目標：4:30:00  策略：負分段  平均配速    │
│                                          │
│  公里分段表（2 欄式）                    │
│  ┌─────────────┬─────────────┐          │
│  │ 1  6:30 6:30│ │22 5:202:14 │          │
│  │ 2  6:25 12:55│ │23 5:202:19│          │
│  │ ...         │ │ ...        │          │
│  └─────────────┴─────────────┘          │
│                                          │
│  高度圖（小圖，5cm 高）                  │
│  ~~~~~~~~~~~~                            │
│                                          │
│  重要提醒                                │
│  · 每 5K 補充水分                        │
│  · 30K 後注意配速變化                    │
└──────────────────────────────────────────┘
```

### 6.3 技術實作方式

#### 6.3.1 PWA 支援

```javascript
// manifest.json 需加入 print 相關設定
{
  "name": "馬拉松配速手環",
  "display": "standalone",
  "orientation": "portrait",
  "prefer_related_applications": true
}
```

#### 6.3.2 列印觸發方式

| 觸發方式 | 說明 |
|---------|------|
| **按鈕點擊** | 配速計劃頁面有「列印」按鈕 |
| **分享選單** | 支援 Android Share Sheet 中的「列印」選項 |
| **URL 參數** | `?print=true` 可直接進入列印預覽模式 |

#### 6.3.3 CSS 列印樣式

```css
/* 螢幕顯示：隱藏列印專用元素 */
@media screen {
  .print-only { display: none; }
}

/* 列印樣式 */
@media print {
  /* 隱藏非必要元素 */
  header, nav, .no-print { display: none; }
  
  /* 強制背景色彩列印 */
  * {
    -webkit-print-color-adjust: exact !important;
    print-color-adjust: exact !important;
  }
  
  /* 確保高度圖和圖表正確列印 */
  canvas, svg { max-width: 100%; }
  
  /* 每頁結尾處理 */
  .page-break { page-break-after: always; }
}
```

#### 6.3.4 Service Worker 快取策略

列印功能需在離線模式下也能正常運作：
- 配速計劃產出後，自動快取到 LocalStorage
- 列印時無需網路，圖表以 Canvas 轉 PNG 方式內嵌
- QR Code 使用 client-side 生成（不需後端 API）

### 6.4 列印預覽功能

| 功能 | 說明 |
|------|------|
| **預覽模式** | 點擊列印前先顯示虛擬預覽 |
| **紙張方向** | 支援直向/橫向切換 |
| **字體大小調整** | 小/中/大三種預設 |
| **黑白模式** | 切換為高對比黑白（節省墨水） |
